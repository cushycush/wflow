# wflow

A macOS Shortcuts-style GUI and workflow engine for Wayland, built on
[wdotool](https://github.com/cushycush/wdotool). Composes input automation +
shell + delays + notifications into shareable `.kdl` workflow files.

---

## Design Context

### Users

General Wayland users — GNOME / KDE / Hyprland — who want a friendly GUI
alternative to shell scripts. Keyboard-first power users who still expect a
product that feels at home alongside other modern desktop apps.

Primary job-to-be-done: _"I keep doing this sequence of things by hand — let
me record it once, name it, and replay it."_

Secondary: sharing workflows as single `.kdl` files (paste on Reddit, commit
to a dotfiles repo).

### Brand Personality

**Modern dark product UI.** Three words: **calm, confident, contemporary.**

This is a real app in a real desktop, alongside Linear / Arc / Raycast /
macOS Shortcuts (dark mode) / modern API clients. Not a website in a window,
not a retro reference, not a bespoke gadget. Thoughtfully-spaced, flat
surfaces, subtle elevation by darkness step, functional color on
category chips, clean sans typography.

Emotional goal: **this is the kind of app someone would use every day and
not notice the chrome** — the UI recedes behind the task.

### Aesthetic Direction

**Visual references:**
- macOS Shortcuts (dark mode) — left sidebar + main pane, rounded
  rectangles, color-coded action categories, clean typography
- Linear — calm dark, subtle elevation, hairline borders, restrained accent
- Raycast — keyboard-first, dense but legible, functional color
- Requestly / Postman — method chips as colored tags, clear hierarchy
- Arc browser settings, Things 3 dark, Notion dark

**Anti-references** (must NOT look like):
- Editorial / magazine layouts (earlier attempt)
- Modular synthesizer / rack aesthetics (earlier attempt)
- Glassmorphism, neon-on-dark, purple-to-blue gradients
- Skeuomorphic hardware details — screws, jacks, patch cables, LEDs
- Dense dashboard SaaS templates (cards everywhere, generic drop shadows)
- Bouncy / elastic animation

### Palette — cool dark with a single warm accent

Committed dark theme. No light mode yet. Neutrals tint very slightly toward
blue (chroma 0.006–0.012 around hue 255). Surfaces step by lightness, not
by border — a clean modern product UI trick.

```
/* Surfaces — background → surface → raised, by lightness */
bg          oklch(0.17 0.010 260)    /* window ground */
surface     oklch(0.20 0.010 260)    /* sidebar, cards, main panel */
surface-2   oklch(0.24 0.012 260)    /* hovered / elevated row */
surface-3   oklch(0.28 0.012 260)    /* input wells, pressed state */
line        oklch(0.30 0.010 260)    /* hairline 1px border */
line-soft   oklch(0.25 0.010 260)    /* quieter divider */

/* Text */
text        oklch(0.95 0.004 260)    /* primary, not pure white */
text-2      oklch(0.72 0.008 260)    /* secondary */
text-3      oklch(0.54 0.010 260)    /* tertiary / disabled */

/* Accent — a single warm amber-ish orange (Raycast-adjacent, not Apple blue,
   differentiates us from the Linear/Apple reflex) */
accent      oklch(0.74 0.15 60)
accent-hi   oklch(0.82 0.14 65)      /* hover */
accent-lo   oklch(0.62 0.17 55)      /* pressed */
accent-dim  oklch(0.34 0.08 58)      /* disabled / track */

/* Semantic — success / warn / error are functional, not decorative */
ok          oklch(0.72 0.16 150)
warn        oklch(0.80 0.15 85)
err         oklch(0.68 0.19 28)
```

**Category colors** — each action kind gets its own chip tint (like HTTP
methods do in API clients). Used only on the small category pill that
labels each action row; never elsewhere.

```
cat-key       oklch(0.72 0.16 285)   /* violet    — KEY   */
cat-type      oklch(0.72 0.16 240)   /* blue      — TYPE  */
cat-click     oklch(0.72 0.16 150)   /* green     — CLICK */
cat-move      oklch(0.72 0.14 200)   /* teal      — MOVE  */
cat-scroll    oklch(0.72 0.14 215)   /* cyan      — SCROLL*/
cat-focus     oklch(0.75 0.14 75)    /* amber     — FOCUS */
cat-wait      oklch(0.58 0.02 260)   /* neutral   — WAIT  */
cat-shell     oklch(0.72 0.16 40)    /* orange    — SHELL */
cat-notify    oklch(0.72 0.16 340)   /* pink      — NOTIFY*/
cat-clip      oklch(0.72 0.14 210)   /* cyan      — CLIP  */
cat-note      oklch(0.54 0.01 260)   /* dim       — NOTE  */
```

Accent discipline: the amber `accent` is used for ONE thing at a time in the
viewport — the primary affordance on the current page (Run, Record arm,
Save) or the selected patch in the sidebar. Category tints only on the chips.
Everything else is neutral.

### Typography

Reflex list still banned. Picks:

- **Hanken Grotesk** — body, UI, headings. A well-built variable grotesque,
  warmer and less corporate than Inter/Geist. Weights 400 / 500 / 600 / 700.
- **Geist Mono** — all technical values, commands, key chords, paths,
  numeric readouts. Paired deliberately for modern-product-UI readability.

Fixed `rem` scale, five steps:

```
--text-xs: 11px    labels, small-caps category chips
--text-sm: 13px    secondary UI text
--text-base: 14px  primary body
--text-md: 16px    section titles
--text-lg: 20px    page titles
--text-xl: 28px    display (rarely used)
```

Two fonts, period. No display/handwriting/industrial faces.

### Design Principles

1. **Surfaces step by lightness.** The hierarchy bg → surface → surface-2 is
   created by brightness alone, not by borders or shadows. A 1px `line`
   hairline is the strongest divider we draw; beyond that, just change the
   fill.

2. **Rounded, but not bubbly.** Every container, row, button, input uses
   `8px` radius. Buttons use `6px`. We don't vary this per-component —
   consistency over cuteness.

3. **Flat, not skeuomorphic.** No gradients on surfaces, no embossed edges,
   no drop shadows unless something is elevated above everything (dialog
   backdrop only). No photo textures.

4. **Category color is functional.** Each action kind has a single color
   tint used only on its 2px×chip-height color dot + category label inside
   the action card. Nowhere else. The accent amber is orthogonal — it's
   the "active action" signal.

5. **Type hierarchy > visual weight.** Titles are 20px Hanken Grotesk 600.
   Body is 14px 400. Values use Geist Mono at 13px. These three things do
   most of the work; don't reach for decorative typography.

6. **Hover is subtle, selection is clear.** Hover raises by one surface
   step. Selected state adds a left 2px accent bar (yes the "stripe
   border" pattern — this is one of the rare places it's the right tool:
   a persistent single-pixel-wide indicator on a list item to show what's
   open, exactly like macOS source lists, VS Code, and every modern IDE).
   Accent bar width ≤ 2px satisfies the skill's anti-pattern rule.

### Motion

Qt Quick provides native declarative animation — use it, but restrainedly.

- Page transitions: `StackView` push/pop at 160ms `Easing.OutCubic`
- Action row expand: `Behavior on height` at 180ms `OutCubic`
- Hover: no animation; instant color swap. Hover animation is a web tic.
- Recording state: the REC button pulses opacity 1.0 ↔ 0.7 at 1.1s
  when armed, steady when recording.
- `prefers-reduced-motion` equivalent: add a `reduceMotion` setting that
  zeroes all durations.

### Accessibility

- Contrast ≥ AA for every text-on-surface combo.
- Full keyboard nav — every interactive element reachable. Tab order
  matches visual order.
- Focus ring: 2px accent outline with 2px offset on the focused element.
- Qt font rendering handles the subpixel / hinting correctly by default.

## Stack

- **Qt 6.11 + Qt Quick (QML)** — UI
- **Rust** — engine (actions, runner, store, recorder, KDL serde)
- **cxx-qt 0.8** — Rust ↔ Qt bridge (Rust types as QObjects)
- **CMake** — build driver; invokes both Qt's moc/rcc/qmltyperegistrar and
  cargo under the hood

One binary. No IPC, no webview. Engine crate is UI-agnostic enough that a
future `wflow-engine` daemon could reuse it behind a D-Bus interface.
