# Packaging

## Arch Linux (AUR)

Two PKGBUILDs in `aur/`:

- `aur/wflow/` — the **release** package. Pulls the v$pkgver release
  tarball from GitHub. Bump `pkgver` and refresh `sha256sums` (replace
  `SKIP` with `updpkgsums`'s output) for each new release.
- `aur/wflow-git/` — the **VCS** package. Always builds the current
  `main`. `pkgver()` derives the version from `git describe`.

### Local build / install

From the repo root:

```sh
cd packaging/aur/wflow-git
makepkg -si
```

`makepkg -s` resolves missing `depends` and `makedepends` via pacman.
`-i` installs after a successful build.

### Submitting to the AUR

The AUR keeps each package in its own git repo (separate from this
repo). Workflow:

```sh
# One-time clone (replace with your AUR ssh remote).
git clone ssh://aur@aur.archlinux.org/wflow.git aur-wflow

# Each release:
cp packaging/aur/wflow/PKGBUILD aur-wflow/PKGBUILD
cd aur-wflow
updpkgsums
makepkg --printsrcinfo > .SRCINFO
git add PKGBUILD .SRCINFO
git commit -m "wflow $(grep -m1 ^pkgver= PKGBUILD | cut -d= -f2)"
git push
```

Same flow for `wflow-git`, swapping the path.

### What gets installed

Both PKGBUILDs install:

- `/usr/bin/wflow` — the binary
- `/usr/share/bash-completion/completions/wflow`
- `/usr/share/zsh/site-functions/_wflow`
- `/usr/share/fish/vendor_completions.d/wflow.fish`
- `/usr/share/man/man1/wflow.1` plus one page per subcommand
  (`wflow-run.1`, `wflow-list.1`, …) — generated by `wflow man --output`
- `/usr/share/doc/wflow/{README,KDL}.md`
- License files at `/usr/share/licenses/wflow/` (when present in the
  source tree)
