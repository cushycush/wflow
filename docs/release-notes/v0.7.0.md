# wflow 0.7.0: bind a chord, fire a workflow

The daemon is awake. wflow now binds keyboard chords to workflows on
KDE Plasma 6, GNOME 46+, Hyprland, and Sway. Add a `trigger { chord "ctrl+alt+t" }`
block to a workflow's KDL, run `wflow daemon`, and the chord fires the
workflow.

If you've been waiting for AutoHotkey on Linux or for the
keyboard-trigger half of macOS Shortcuts, this is the part of wflow
you've been waiting for.

## How it picks the right backend

The daemon probes three paths at startup, in priority order:

1. **GlobalShortcuts portal** (`org.freedesktop.portal.GlobalShortcuts`).
   Plasma 6 and GNOME 46+ both ship it. The first time you bind a
   chord, the desktop pops a consent dialog showing what you're
   asking for. Accept it once and the chords are live for the
   session.

2. **Hyprland IPC.** Hyprland doesn't ship the portal yet, so the
   daemon writes `bind` directives via
   `$XDG_RUNTIME_DIR/hypr/$HIS/.socket.sock` pointing back at
   `wflow run <id> --yes`.

3. **Sway IPC.** Same shape as Hyprland, over `$SWAYSOCK` using the
   i3 IPC RUN_COMMAND protocol:
   `bindsym MODS+KEY exec wflow run <id> --yes`.

Whatever's available wins. On a Plasma session that the portal can
reach, the daemon uses the portal. On Hyprland it uses the IPC
fallback. You don't pick.

## Hot reload

Edit a workflow's KDL while the daemon's running. Rebind the chord,
add a new trigger, drop one. The daemon watches the workflow library
via inotify, diffs the chord set against what's currently bound, and
unbinds-then-rebinds the deltas. No daemon restart. No `hyprctl reload`.

Caveat: the GlobalShortcuts portal binds shortcuts once per session
by spec, so portal mode (Plasma 6, GNOME 46+) does need a daemon
restart to pick up trigger changes. Compositor-IPC mode (Hyprland,
Sway) is fully live-reloadable.

## Auto-start on login

The daemon should be running every time you sign in. AUR, Flatpak,
and distro packages drop a systemd user unit at
`/usr/lib/systemd/user/wflow-daemon.service`. One command and you're
set up for every login:

```sh
systemctl --user enable --now wflow-daemon
```

The unit ties to `graphical-session.target`, so the daemon starts
when KDE / GNOME / Hyprland / Sway come up and stops on logout.
`Restart=on-failure` handles crashes. `journalctl --user -u wflow-daemon`
is where the logs land.

If you installed via `cargo install --path .`, edit `ExecStart` in the
unit to use an absolute path. Systemd's user environment doesn't
always inherit your shell's PATH, so `~/.cargo/bin/wflow daemon`
explicit beats `wflow daemon` lookup-fingers-crossed.

## Single-instance lock

Two `wflow daemon` invocations can't both bind the same chords. The
first writes a pidfile to `$XDG_RUNTIME_DIR/wflow/daemon.pid`. The
second sees the pidfile, checks `/proc/$pid` to confirm the first is
still alive, and exits with "already running (pid N)". Crashed
daemons leave a stale pidfile that the next start treats as
recoverable.

## Quiet engine improvement

wdotool-core moved 0.4 → 0.5 along with this release. The wlroots
backend now does a `queue.roundtrip()` after every input op (key,
type, mouse-move, mouse-button, scroll), so a fast wflow process
can't exit and destroy its virtual devices before the compositor
finishes processing in-flight events. Caught silent input drops on
wlroots that nobody knew were happening. Existing workflows on
Hyprland and Sway should feel slightly more reliable on tight
sequences, especially long type / key chains.

## What's not in 0.7

A few things AHK users will ask about that aren't here:

- **Hotstrings (text expansion).** The bit of AHK where typing `btw`
  expands to "by the way" needs a global keyboard monitor on top of
  the daemon. Different problem from chord binding. Post-0.7.
- **Per-window conditional hotkeys.** "When Firefox is foreground,
  ctrl+t opens my reading list, otherwise the default behaviour."
  Same release.
- **Triggers tab in the GUI.** Author triggers in the workflow's KDL
  for now. The GUI Triggers tab needs the eventual D-Bus surface
  for the daemon, which isn't shipping in 0.7.
