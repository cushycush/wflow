# wflow 0.5.0: pick your skin

This release is about how wflow looks. The app now ships with two brand
palettes side by side, picks one with you on the first run, and lets
you flip between them any time from Settings. Pick the one that fits
the room.

If you tried 0.4.x and the cool slate-blue felt right, nothing changes
unless you want it to. If you want the warm cream paper from
[wflows.io](https://wflows.io), it's a single tap on first launch.

## Two skins

**Warm Paper** is the wflows.io identity. Cream surfaces around hue
55, a coral accent at hue 25, ink-tinted category chips that sit
calmly against the paper. It's the palette the marketing site wears,
now the default in the app.

**Cool Slate** is the original brief: slate-blue surfaces at hue 260
with low chroma, a warm amber accent at hue 60. Saturated category
chips. The 0.4.x look, exactly, kept around for anyone who liked it
where it was.

Both palettes carry full light + dark variants. Theme + palette flip
independently, so any of the four combinations (warm light, warm dark,
cool light, cool dark) is a real surface you can live in.

The palette is stored in `state.toml` next to your theme preference
and survives restart.

## The tutorial picks for you

The first-run coach-mark tour now opens with a "Pick your look" step.
Two side-by-side preview tiles, each one rendering its own palette
regardless of what's active. Tap one and the rest of the tour reskins
live as it advances. Existing users get the new step on next launch
(the tour key bumped to `intro_tour_v3`).

If you already dismissed the v2 tour and don't want to walk through it
again, Settings → Palette has the same two-segment toggle.

## Editor canvas, polished

While the brand work was open, the editor canvas got a round of
small fixes that had been nagging:

- **Connector ports went flat.** The radial halo plus offset white
  highlight gave them a 3D feel that didn't match anything else on the
  canvas. Each port is now a single coral disk with a hairline ring,
  reading from the same family as `CategoryIcon`.
- **Wires picked up the active palette.** The marching dashes used to
  be a hardcoded cool cyan that fought the warm palette. They now use
  `Theme.lineStrong` so they sit as quiet structure on either skin.
- **Chips and toolbar glyphs match.** Step cards on the canvas, mini
  steps in the explore preview, and chips in the toolbar palette all
  render the same kind glyph at the same optical weight. The chevron
  on `shell` looked oversized on the canvas while reading perfectly in
  the toolbar; both go through `CategoryIcon` now and pick up its
  per-kind ratio system.
- **Inner step icons inside Repeat containers got their proportions
  back.** They were rendering at `size: 14`, below the threshold where
  the per-kind ratio actually does anything, so chevron / timer /
  pilcrow all collapsed to the same dense 10px floor. Bumped to 18.
- **Dot grid is editor-only.** The graph-paper backdrop used to live
  app-wide. It now renders only behind the workflow canvas, so
  Library, Explore, Record, and Settings sit on a clean `Theme.bg`.
  The dots themselves use `Theme.text2` at 10% alpha so they read on
  either palette without per-palette branching.

## Buttons, cards, and the nav pill

- Primary and Secondary buttons are pill-shaped now, matching the
  wflows.io button language. Secondary thickens its border to
  `lineStrong` on hover.
- Library cards bump from `radiusMd` to `radiusLg` and swap their
  hover border from a category-color wash to `lineStrong`, the same
  treatment wflows.io uses for `.workflow-card`.
- The selected tab in the floating nav pill no longer wears a coral
  focus ring. Click was forcing keyboard focus, which woke up
  `FocusRing` on top of the existing accent-wash fill. Selection now
  reads from the fill alone; keyboard tab still works as expected.
- `CategoryChip` is pill-shaped with a hairline border. Quieter,
  closer to wflows.io `.wf-trigger`.

## Fonts back where they were

A short experiment swapped the font pair to the wflows.io brand
faces (Boska serif + Supreme sans). They read poorly at the dense UI
sizes wflow uses, especially in the inspector and palette dock. Back
on Hanken Grotesk + Geist Mono, the 0.4.x pair. The `familyDisplay`
token stays around as a separate hook so a future heavier display
face can land without churn.

## One real bug

A static `accentWash` token I added during the brand experiment
collided with the existing `accentWash(alpha)` helper of the same
name. QML's binding tracker silently picked the property over the
function, every callsite doing `Theme.accentWash(0.18)` returned the
fixed light-mode color regardless of theme, and selected folder rows,
selected wires, and menu hover items all rendered a cream tone in
dark mode. Renamed the static token to `accentDim`, the helper now
resolves correctly, and the design-context section of `CLAUDE.md`
carries a one-line note so the trap doesn't get re-set.

## Compatibility

`state.toml` gains a `palette` field. Existing files load fine: the
serde default kicks in and you get warm paper. Already on 0.4.x and
want to stay on the original look? Open Settings, flip Palette to
Cool Slate, done.

No KDL changes. No engine changes. Workflows recorded under 0.4.x
run identically.
