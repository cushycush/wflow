# wflow 0.6.0: conditionals get a real false branch

`when` and `unless` now accept an `else { ... }` block. The canvas
draws the no-side as a parallel column or row across every layout.
The inspector grew a FALSE BRANCH section so authoring matches the
engine. Wire routing got cleaner along the way.

If you've ever wanted a `when window="X" else { ... }` shape for the
cases your workflow has to handle when X isn't open, this is the
release.

## else, in KDL

The new block lives inside the parent `when` / `unless`:

```kdl
when window="Slack" {
    focus "Slack"
    key "ctrl+k"
    type "{{channel}}"
    key "Return"

    else {
        shell "slack &"
        wait-window "Slack" timeout="20s"
        notify "launched Slack" body="re-run to use it"
    }
}
```

Engine semantics: the predicate runs once; if true, `steps` runs and
`else` is skipped; if false, `else` runs and `steps` is skipped. An
empty `else { }` collapses to no-op (same as the historical "skip
silently when the predicate is false" behaviour, so existing files
round-trip unchanged).

The parser rejects two structural mistakes with clear errors:

- More than one `else` block per parent.
- Steps after the `else` block (the `else` must come last).

## else, on the canvas

Every layout draws both branches now. Wires fork from the parent
toward each branch and rejoin at the next-top.

**Vertical** fans yes-cards to the right of the conditional and
no-cards to the left, both starting at the conditional's vertical
mid. Wires exit the parent's left and right sides — the natural
adjacent-edge diagonal, no loop-around lobe.

**Horizontal** drops yes-cards below the parent and no-cards above.
The branch rows start past the conditional's right edge with a gap
so a wide conditional doesn't bleed into the row visually. Wires
fork off the parent's top and bottom.

**Grid** aligns every parent in a row on a single baseline. No-stack
reaches up from the parent's top, yes-stack reaches down from the
parent's bottom. Adjacent non-conditional cells in the row sit on
the same baseline, so inter-cell wires run horizontally between
cells without threading through any branch stack.

**Smart Tidy** splits each conditional cell with no-stack on the
left and yes-stack on the right. Each Smart column pre-computes a
leftPad (max no-column width across cells in the column) so every
parent in the column lands at the same X — keeps inter-cell
top-to-bottom wires running through one vertical line.

## else, in the inspector

The inspector picks up two changes when you click a conditional.

The existing INNER STEPS section renames to **TRUE BRANCH** so it
sits symmetrically with the new **FALSE BRANCH** section right
below. Both sections support add / delete with the same UI shape.
Repeat keeps INNER STEPS — there's no true / false split there.

## Wire routing fixes

Same-row back-flow used to do a sweeping right-out / left-in lobe,
which on a vertical-layout conditional showed up as a pointless
U-turn dive below the parent before climbing back up to the
no-side card. Now back-flow goes left-out / right-in: clean
horizontal diagonal between adjacent edges. The forward case is
unchanged.

The router also dropped a stale rule that switched same-row
back-flow to vertical when the target's centre was below the
source's centre. That rule fired exactly when a conditional's
first no-side card sat at the parent's mid-y in vertical layout,
so the no-side wire was the one path it broke. Now any same-row
back-flow uses horizontal back-flow regardless of small Y deltas.

## Reset zoom + first-load auto-fit

Two viewport quality-of-life fixes that fell out of dogfooding the
canvas changes.

**Reset zoom** (the percentage chip) now preserves the world point
at the viewport centre through the zoom-to-1.0 transition. The
button used to pass raw contentX/Y across the zoom delta, but
contentWidth is `world * zoom` so contentX is in scaled units;
switching zooms across that hand-off shifted the viewport off the
cards. Now we convert the viewport centre to world coords first,
set zoom to 1.0, then position content so the same world point
lands at the viewport centre.

**First-load auto-fit** snaps the viewport to all cards on every
fresh workflow open. Used to be gated on "no positions yet", which
only triggered for the very first save of a new workflow; reopening
an authored workflow landed the user at scene origin with the cards
out of view. Now a 120ms timer waits for card heights to publish
via onHeightChanged, then zooms to fit. A second pass on
cardHeights changes within 500ms re-arms the timer if measurements
arrive in waves, so the final fit reflects the real layout even on
slow renders.
