# wflow 0.4.0: the editor grows up

The biggest release wflow has shipped. The editor stops being a step
list with an inspector and becomes a real node-graph workspace. Free
positioning, branch shapes, repeat containers, multi-select and
marquee, undo and redo, group rectangles, a step-by-step debugger
with per-iteration pulse, fragment imports via `use NAME`, and a
first-run tour that covers all of it. First release where the GUI
feels like a first-class way to build workflows rather than a
friendly skin over the KDL file.

If you tried 0.3.x and bounced because the editor felt like a list
view with extra steps, this is the version to come back for.

## The canvas

Every step is a card on a free-positioning canvas. Wires auto-route
between consecutive steps, picking entry and exit sides per pair so
the diagram reads cleanly without you nudging anything. Conditional
steps render as branch decision shapes with explicit yes / no
outputs; their inner steps surface as siblings on the canvas with
fork-and-rejoin wires drawn for you. Repeat is a container card with
an inline strip of inner steps, since loops are about sequencing,
not branching.

Drag chips from the palette dock on the left to spawn steps anywhere.
The right-side tool dock has four "Tidy" buttons. The new default,
**Smart Tidy**, sweeps column counts and picks the layout that keeps
cards readable at the closest-to-1.0 zoom; Vertical, Horizontal, and
Grid stay for users who want a specific shape.

Wheel zooms around the cursor. Drag empty canvas to pan. Card
positions persist alongside the workflow file, so the layout you
arrived at survives reopening.

## Multi-select, undo, group rectangles

Shift- or ctrl-click cards to add to the current selection.
Shift- or ctrl-drag empty canvas to lasso. Selected cards move
together, delete together, and selection highlights live as the
marquee crosses cards. The Delete key is context-aware ("Delete
step" / "Delete N steps" / "Delete workflow").

Undo and redo are wired up. Ctrl+Z / Ctrl+Shift+Z, with debouncing
so a fast double-tap doesn't leapfrog past the state you wanted to
return to.

Alt-drag draws a coloured group rectangle behind cards as a visual
annotation. Right-click for a swatch picker, double-click for an
inline comment. The engine ignores them. They exist for the "this
section is the build half" / "this section is the deploy half"
mental scaffolding.

## Step-by-step debugging

The new ⏯ Debug button next to ▶ Run pauses the engine between every
action. **Step** advances one step, **Continue** resumes, **Stop**
bails. The active card pulses while it's firing; each step's status
dot in the header settles to green / red / grey on outcome.

Repeat shines in debug mode. Every iteration's inner step gets its
own pulse, so you can watch the loop count up instead of guessing
how many laps the engine has done.

## Reuse via imports

A `use NAME` step splices in another workflow file. The card carries
an Open → button that loads the fragment in a new editor tab, so
you can edit imported pieces without leaving the workspace. Tabs
along the top of the editor preserve per-doc state (selection,
breadcrumb, save state) when you switch between them.

## The tour, refreshed

The coach-mark tour now covers debug, multi-select + groups, and
imports. Returning users get one auto-replay with the new content;
the storage key bumped from `intro_tour` to `intro_tour_v2` so
existing installs don't silently miss the new steps. Settings still
has a "Replay tour" button when you want it again.

## Eight bundled templates

Morning sync joins the seven 0.3.x templates as the new showcase
example. It exercises every action category plus when, repeat, use,
and groups in a single file: pull latest, run tests, post a standup
message via a fragment import. The other seven (dev-setup, screenshot
and share, daily standup, loop tab thru, if-vpn-then,
flaky-deploy-trigger, record-replay-export) are unchanged.

## Explore is hidden, briefly

The Explore tab is gated off in 0.4.0 (`Theme.showExplore = false`).
The catalog is mock data until the wflows.com integration lands, and
shipping mock data behind a "Browse community workflows" pill felt
like the wrong promise. Explore comes back on as part of the v1.0
milestone alongside sign-in, the deeplink confirm dialog, and a
detail drawer wired to live data. `BACKLOG.md` restructures around
that: every wflows.com integration item now lives under a single
"v1.0" header.

## Smaller things worth knowing

- Status dots flash transitions, so ok / error / skipped lands
  visibly mid-run instead of just snapping in.
- Card hover controls (× delete, rewire button) stay up while the
  cursor is anywhere on the card, not just on the dragArea. They
  used to vanish the moment you tried to click them.
- Marquee selection works at any zoom + scroll position. The earlier
  attempt projected coordinates by hand and drifted whenever the
  canvas was scrolled; now everything lives in world-local space.
- Group rectangle hover lifts the border and fill alpha so you can
  tell which group your cursor is over. Resize uses
  `DragHandler.translation` instead of mouse-position feedback, so
  the dimensions don't oscillate as you drag.
- Tooltips dropped from the canvas tool dock and step palette dock.
  Both expand on hover and show their labels inline; the floating
  tooltip on top of that was the same text twice.
- Wire routing went through several rounds. Current rules: vertical
  chains exit the source's bottom, enter the target's top; horizontal
  chains exit right, enter left; rows beat cols (different-row pairs
  go vertical even if the X delta is wider). Back-flow wires check
  the target's clearance: cards above target → enter from the bottom;
  X ranges overlap → use the natural top→bottom edges instead of the
  column-wrap lobe; otherwise the strict bottom→top stays. The
  result reads as flow continuation in every layout, not as a wire
  picking the geometric shortest path.

## Engine changes worth flagging

Two new bridge signals power the run-feedback dots. `step_done` now
carries `step_id` alongside `index` and `status`, which lets the UI
attach status to inner steps inside a Repeat container that don't
have a top-level card of their own. `step_started(index, step_id)`
is a new unconditional signal that fires on every StepStart, even
when the same id is set twice in a row. Without it, cxx-qt's
setter-side dedupe meant the inner pulse only flashed once across
all iterations of a loop.

Both are covered by new engine tests that watch a 3-iteration repeat
fire the right number of starts and dones with stable ids.

## Upgrading

`paru -Syu wflow-bin` on Arch via the AUR. The binary is the same one
the GitHub release attaches. From source: `cargo install --path .
--locked` against this tag. KDL workflows from 0.3.x load and re-save
unchanged. The canvas-positions block is new but optional; existing
files get default placements that you can rearrange and persist.

## What's next

0.4.x is the new baseline. v1.0 means full wflows.com integration:
Explore back on with a real catalog, sign-in via Better Auth, deeplink
confirm dialog, and the detail drawer wired to live data. The v0.5
daemon work (real OS hotkey trigger binding via portal / Hyprland IPC)
lands in parallel.

Bug reports, PRs, and "this should be one of the bundled templates"
suggestions all welcome.
