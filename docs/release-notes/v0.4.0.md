# wflow 0.4.0 — the editor grows up

The headline of 0.4.0 is the editor. What started as a step list with an
inspector is now a real node-graph workspace: you place cards, the
engine flows between them, you can debug it. It's the first release
where the GUI feels like a first-class way to build workflows rather
than a friendly skin over the KDL file.

## The canvas

Every step is a card you can drop anywhere on an effectively-infinite
canvas. Wires auto-route between consecutive steps, picking entry and
exit sides per pair so the diagram reads cleanly without you nudging
anything. Conditional steps render as branch decision shapes with
explicit yes / no outputs; their inner steps surface as siblings on the
canvas, with fork-and-rejoin wires drawn for you. Repeat is a container
card with an inline strip of inner steps, since loops are about
sequencing, not branching.

Drag chips from the palette dock on the left to spawn steps anywhere.
Three "Organize" buttons in the top-right re-arrange every card into a
vertical / horizontal / grid layout in one shot. Otherwise, positions
are sticky — adding a step from the rail or resizing the window won't
reflow what's already there.

Wheel zooms around the cursor, drag empty canvas to pan, and the layout
survives reopening the workflow because positions persist alongside the
KDL.

## Multi-select, undo, groups

Shift- or ctrl-click cards to add to the current selection. Shift- or
ctrl-drag empty canvas to lasso. Selected cards move together, delete
together, and the Delete key is now context-aware ("Delete step" /
"Delete N steps" / "Delete workflow" depending on what's selected).

Undo and redo are wired up. Ctrl+Z / Ctrl+Shift+Z, with single-click
debouncing so a fast double-tap of Undo doesn't leapfrog past the
state you wanted to return to.

Alt-drag draws a coloured group rectangle behind cards as a visual
annotation. Right-click for a swatch picker, double-click for an
inline comment. The engine ignores them — they're for the
"this section is the build half" / "this section is the deploy half"
mental scaffolding. Esc commits the comment now (used to lose the
text you just typed).

## Step-by-step debugging

The new ⏯ Debug button next to ▶ Run pauses the engine between every
action. **Step** advances one step, **Continue** resumes, **Stop**
bails out. The active card pulses while it's firing; the green status
dot in its header settles to green / red / grey on each leaf as ok /
error / skipped.

Repeat is fun in debug mode: each iteration's inner step gets its own
pulse, so you can watch the loop count up. Notes are skipped in
debug — the debugger walks only steps that actually execute.

## Reuse via imports

Drop a `use NAME` step to splice in another workflow file. The card
gets an Open → button that loads the fragment in a new editor tab, so
you can edit imported pieces without leaving the workspace. Tabs along
the top of the editor preserve per-doc state (selection, crumb, save
state) when you switch between them.

## The first-run tour, refreshed

The coach-mark tour now covers debug, multi-select + groups, and
imports. Returning users get one auto-replay with the new content; the
storage key bumped from `intro_tour` to `intro_tour_v2` so existing
installs don't silently miss the new steps. Settings still has a
"Replay tour" button when you want it again.

## Explore is hidden, briefly

The Explore tab is gated off in 0.4.0 (`Theme.showExplore = false`).
The catalog is mock data until the wflows.com integration lands, and
shipping mock data behind a "Browse community workflows" pill felt
like the wrong promise. Explore comes back on as part of the v1.0
milestone alongside sign-in, the deeplink confirm dialog, and a
detail drawer wired to live data.

## Smaller things worth knowing

- Status dots flash transitions, so ok / error / skipped lands
  visibly mid-run instead of just snapping in.
- Card hover controls (× delete, rewire button) stay up while the
  cursor is anywhere on the card, not just on the dragArea — used to
  vanish the moment you tried to click them.
- Drag-from-folder-onto-card no longer corrupts library positions on
  the rare-but-real case where the drop hit a stale index.
- Marquee selection works at any zoom + scroll position. The earlier
  attempt projected coordinates by hand and drifted whenever the
  canvas was scrolled; now everything lives in world-local space.
- Hover state lifts a group rectangle's border + fill alpha so you
  can tell which group your cursor is over. Resize uses
  `DragHandler.translation` instead of mouse-position feedback so the
  dimensions don't oscillate as you drag.

## Engine changes worth flagging

Two new bridge signals power the run-feedback dots:

- `step_done` now carries `step_id` alongside `index` and `status`.
  This lets the UI attach status to inner steps inside a Repeat
  container that don't have a top-level card of their own.
- `step_started(index, step_id)` is a new unconditional signal — fires
  on every StepStart, even when the same id is set twice in a row.
  Without it, cxx-qt's setter-side dedupe meant the inner pulse only
  ever flashed once across all iterations of a loop.

Both are covered by new engine tests that watch a 3-iteration repeat
fire the right number of starts/dones with stable ids.

## Upgrading

`paru -Syu wflow-bin` if you're on Arch with the AUR; the binary is
the same one the GitHub release attaches. From-source: `cargo install
--path . --locked` against this tag. KDL workflows from older
releases load and re-save unchanged — the canvas-positions block is
new but optional, and existing files just get default placements
that you can rearrange and persist.

## What's next

v0.4.x is the new baseline; v1.0 = full wflows.com integration. That
means the Explore tab back on, sign-in via Better Auth, the deeplink
confirm dialog, and the detail drawer wired to real catalog data.
The v0.5 daemon work (real OS hotkey trigger binding via portal /
Hyprland IPC) lands in parallel.

Bug reports, PRs, and "this should be one of the bundled templates"
suggestions all welcome.
